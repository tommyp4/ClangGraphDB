## 🔍 Analysis & Investigation
There is a case-sensitivity mismatch in domain naming and node retrieval within the graphdb pipeline.
- `internal/rpg/naming.go` generates domain names starting with lowercase `domain-` and `domain-generic-`.
- `internal/query/neo4j_batch.go` queries for unnamed features using a case-sensitive `STARTS WITH 'Domain-'` clause (as well as `STARTS WITH 'Feature-'`).

As a result, domain nodes generated by `naming.go` are not picked up by the enrichment pipeline because they do not match `Domain-*`.

**Impact Analysis (Findings from `GRAPHDB_TOP_LEVEL_DOMAINS.md`):**
A recent structural analysis of the `trucks-v2` project graph highlighted the severe impact of this case mismatch. Because the existing nodes are named in lowercase, the enrichment pipeline currently misses **28 critical top-level domains**, including:
1.  **The Application Domain (`domain-app`):** The primary frontend cluster encompassing 100 functions.
2.  **27 Generic Backend Domains (`domain-generic-*`):** Massive clusters containing all core business logic, such as Settlement Processing (322 functions), Data Extraction (183 functions), Compliance (133 functions), and Persistence workflows (120 functions).
3.  **Anomalous Nodes:** The analysis also identified a root node named `"Top Feature"` (a remnant from a test) that likewise evades enrichment due to exact-match filtering.

Because these critical clusters reside as lowercase strings, the `GetUnnamedFeatures` query silently bypasses them, leaving the system's most crucial architectural nodes unenriched with proper semantic names and summaries.

To fix this, we need to:
1. Update `GetUnnamedFeatures` in `internal/query/neo4j_batch.go` to use case-insensitive matching (`toLower(coalesce(n.name, '')) STARTS WITH 'domain-'`). This ensures backward compatibility and crucially allows the enrichment of the 28 massive clusters already persisted in the graph. We should also include logic to capture the `"Top Feature"` test remnant.
2. Update `GenerateDomainName` in `internal/rpg/naming.go` to output standardized `Domain-` and `Domain-Generic-` prefixes for consistency and readability when viewing newly generated nodes.

## 📝 Implementation Plan

### Prerequisites
None

### Step-by-Step Implementation

#### Phase 1: The Harness (Testing case-insensitive matching)
1.  **Step 1.A:** Update Neo4j query tests.
    *   *Action:* Modify `internal/query/neo4j_batch_test.go`. Add mock nodes with lowercase `domain-*`, uppercase `Domain-*`, lowercase `feature-*`, and the literal `Top Feature` node to the `setupQuery`. Assert they are all retrieved by `GetUnnamedFeatures`.
    *   *Goal:* Ensure the test fails initially before making the code change (Red phase).

#### Phase 2: The Implementation (Case-insensitive matching)
1.  **Step 2.A:** Update Neo4j query.
    *   *Action:* Modify `internal/query/neo4j_batch.go` in the `GetUnnamedFeatures` method. Change the `WHERE` clause to:
        ```cypher
        WHERE coalesce(n.name, '') = '' OR toLower(coalesce(n.name, '')) STARTS WITH 'feature-' OR toLower(coalesce(n.name, '')) STARTS WITH 'domain-' OR toLower(coalesce(n.name, '')) = 'top feature'
        ```
    *   *Detail:* This makes the fallback processing robust to any case variations, ensuring all 28 major top-level domains (e.g., `domain-app`, `domain-generic-*`) and anomalous nodes are enriched.
2.  **Step 2.B:** Verify Neo4j query.
    *   *Action:* Run `go test ./internal/query -run TestNeo4jBatch` to ensure tests now pass.

#### Phase 3: The Implementation (Standardizing Domain naming)
1.  **Step 3.A:** Update Naming tests.
    *   *Action:* Modify `internal/rpg/naming_test.go`. Update the `expected` string fields in the test cases to start with uppercase `Domain-` and `Domain-Generic-`.
    *   *Goal:* Assert the new expected behavior of `GenerateDomainName` (Red phase).
2.  **Step 3.B:** Update Domain generation logic.
    *   *Action:* Modify `internal/rpg/naming.go` inside the `GenerateDomainName` method. Return strings prefixed with `"Domain-"` and `"Domain-Generic-"` instead of their lowercase variants.
    *   *Detail:* Update `return "domain-" + strings.ToLower(base)`, `return "domain-" + term`, and `return "domain-generic-" + GenerateShortUUID()` to return their respective capitalized prefixes.
3.  **Step 3.C:** Verify Naming logic.
    *   *Action:* Run `go test ./internal/rpg -run TestGenerateDomainName` and ensure tests pass.
