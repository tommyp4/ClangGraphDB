# Campaign 11.6: State-Driven Semantic Naming (Eliminating Magic Strings)

## 🎯 Goal
Replace the frail, magic-string-based (`STARTS WITH "Domain-"`) feature naming system with a robust, schema-driven approach. 

During the RPG domain clustering phase, domains are generated and need names. Currently, the system relies on string prefixes to determine if a domain needs semantic enrichment (e.g. naming it `domain-generic-<UUID>` to flag it for the LLM). This creates a disconnected data pipeline where valid top-level domains are dropped, missed by queries, or caught in "Semi-Named Purgatory" (having a name but no summary). 

This campaign decouples Node Identity (UUIDs) from Semantic State (Name/Summary properties), ensuring every discovered structural cluster is reliably enriched by the LLM.

## 📋 Key Deliverables
- [x] **Decouple ID from State:** Remove all `"Feature-"` and `"Domain-Generic-"` string injections from `internal/rpg/naming.go` and clusterers. Use raw UUIDs for IDs. ✅
- [x] **State-Driven Enrichment:** Refactor the enrichment pipeline to use explicitly empty properties (`n.name = ""` or `n.summary = ""`) to determine if a node needs processing, rather than parsing its name. ✅
- [x] **Data Pipeline Fix:** Update `GetFunctionMetadata` to retrieve the `n.name` property, re-enabling the `findDominantTerm` fallback logic. ✅
- [x] **Eliminate Magic Strings:** Remove `STARTS WITH 'Feature-'` and `STARTS WITH 'Domain-'` from Cypher queries. ✅

## 🛠 Implementation Plan

### Phase 1: Fix Data Visibility (The Root Cause) ✅
The initial naming fallback (`findDominantTerm`) is currently set up to fail because the data it needs isn't loaded from Neo4j.
1.  **Action:** Modify `internal/query/neo4j_batch.go`.
    *   In the `GetFunctionMetadata` method, update the Cypher query to return `n.name as name`. ✅
    *   Parse the `name` property from the result record and add it to the `graph.Node.Properties` map. ✅
    *   *Goal:* Ensure the clustering engine can actually read function names to generate intelligent fallback names like `Domain-Settlement`. ✅

### Phase 2: Decouple ID and State ✅
If `findDominantTerm` fails, we must explicitly declare the node "Unnamed" rather than inventing a misleading generic name.
1.  **Action:** Modify `internal/rpg/naming.go`.
    *   Change `GenerateDomainName` to return an empty string `""` instead of `"domain-generic-" + GenerateShortUUID()` when `findDominantTerm` fails. ✅
    *   Remove the `Domain-` prefix logic entirely from this utility. ✅
2.  **Action:** Modify `internal/rpg/builder.go`.
    *   When generating the `domainID`, use a strict format: `domain-<UUID>`. ✅
    *   If `GenerateDomainName` returns an empty string, leave the `domainFeature.Name` as `""`. ✅
3.  **Action:** Modify `internal/rpg/cluster_semantic.go`.
    *   When creating cluster keys (which become IDs), strictly use `cluster-<UUID>`. ✅
    *   Do not inject `"Feature-"` into the strings. ✅

### Phase 3: State-Driven Enrichment Logic ✅
Update the query to rely on schema state (empty properties) rather than magic strings.
1.  **Action:** Modify `internal/query/neo4j_batch.go`.
    *   In `GetUnnamedFeatures`, change the `WHERE` clause to: ✅
        ```cypher
        WHERE coalesce(n.name, '') = '' OR coalesce(n.summary, '') = ''
        ```
    *   *Goal:* Catch nodes that have no name, OR nodes that got a name during clustering but never received an LLM-generated summary (The "Semi-Named Purgatory"). ✅

### Phase 4: Validation ✅
1.  **Action:** Run all tests in `./internal/rpg/...` and `./internal/query/...`. ✅

    *   Update `internal/rpg/naming_test.go` to verify it returns `""` instead of `domain-generic-`. ✅
    *   Update `internal/rpg/builder_test.go` to account for the new ID generation. ✅
