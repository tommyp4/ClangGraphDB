## 🔍 Analysis & Investigation
Current analysis confirms that the "Unknown Feature" issue in RPG generation is caused by `rpg/enrich.go` expecting `content` in node properties, which is never populated (except by `vbnet.go`, which does it incorrectly by dumping the whole file).

**Strategy:**
1.  **Shift Left:** Parsers must identify the *boundary* of functions (`start_line`, `end_line`).
2.  **Just-In-Time:** The Enricher will read the source code from disk only when needed for the LLM context window.
3.  **Clean Data:** We explicitly forbid storing source code in the Graph DB (JSONL) to keep the index small and fast.

## 📝 Implementation Plan

### Prerequisites
- `internal/tools/snippet` must be available (Confirmed).

### Step-by-Step Implementation

#### Phase 3.7.1: Parser Refactoring

1.  **Step 3.7.1.A (Tree-sitter Parsers):** Add `end_line` to node properties.
    *   *Target Files:* 
        - `internal/analysis/csharp.go`
        - `internal/analysis/typescript.go`
        - `internal/analysis/java.go`
        - `internal/analysis/cpp.go`
        - `internal/analysis/sql.go`
    *   *Action:* In the `Parse` function loop where nodes are created:
        - Identify the `start_line` assignment (usually `c.Node.StartPoint().Row + 1`).
        - Add `end_line`: `c.Node.EndPoint().Row + 1`.
        - Ensure *no* `content` property is set.
    *   *Verification:* `go test ./internal/analysis/...`

2.  **Step 3.7.1.B (VB.NET Parser):** Remove content blob and track lines.
    *   *Target File:* `internal/analysis/vbnet.go`
    *   *Action:*
        - **CRITICAL:** Remove `Properties["content"]` from the File node creation.
        - Refactor the line-by-line scanning loop to track function start *and* end.
        - *Logic:* 
            - When `Function/Sub` matches: Store `currentFunc` state with `start_line`.
            - When `End Function/Sub` matches: Retrieve `currentFunc`, set `end_line` to current line, create Node, and append to list.
    *   *Verification:* `go test ./internal/analysis/vbnet_test.go`

#### Phase 3.7.2: Enricher Refactoring

1.  **Step 3.7.2.A (Define SourceLoader):** Decouple content reading.
    *   *Target File:* `internal/rpg/enrich.go`
    *   *Action:*
        - Define `type SourceLoader func(path string, start, end int) (string, error)`.
        - Add `Loader SourceLoader` to `Enricher` struct.
        - In `cmd/graphdb/main.go` (or wherever `Enricher` is instantiated), set `Loader = snippet.SliceFile`.
    
2.  **Step 3.7.2.B (Implement Read-on-Demand):** Update `Enrich` logic.
    *   *Target File:* `internal/rpg/enrich.go`
    *   *Action:*
        - Remove the check for `fn.Properties["content"]`.
        - Instead, extract `file`, `line` (start), and `end_line` from `fn.Properties`.
        - Call `e.Loader(file, line, end_line)`.
        - Handle errors gracefully (log warning, skip snippet).
    *   *Verification:* Create a test in `internal/rpg/enrich_test.go` that mocks `Loader` and asserts it is called with correct lines.

#### Phase 3.7.3: Final Verification

1.  **Step 3.7.3.A (E2E Test):**
    *   *Action:* Run the ingestion on a sample project (e.g., `test/fixtures/csharp`).
    *   *Check:* 
        1. `graph.jsonl` does *not* contain `content` fields.
        2. `graph.jsonl` *does* contain `end_line` for functions.
        3. Run the Enricher (if CLI allows) and verify it doesn't crash.
