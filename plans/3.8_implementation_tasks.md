## 🔍 Analysis & Investigation
**Context:**
Currently, `rpg.Builder` uses directory-based discovery (`DirectoryDomainDiscoverer`) to identify domains. This fails to capture "Latent Domains" (cross-cutting concerns scattered across folders).
The goal is to implement "Latent Architecture Recovery" by clustering *all* repository functions using their embeddings, independent of file location, and then grounding these clusters to the file system via Lowest Common Ancestor (LCA).

**Architecture:**
*   **`GlobalEmbeddingClusterer`:** A new component that:
    1.  Uses `EmbeddingClusterer` (K-Means) to partition the entire universe of functions.
    2.  Identifies "Central Nodes" (closest to centroid) for each cluster.
    3.  Uses `Summarizer` (LLM) to generate a semantic name (e.g., "Authentication System") for the cluster.
*   **`Builder.buildGlobal`:** Existing method needs refinement to respect the semantic names generated by the clusterer, instead of overriding them with `GenerateDomainName` (heuristic).
*   **`LCA Grounding`:** Already implemented in `FindLowestCommonAncestor`, but needs to be applied to the global clusters to determine their `ScopePath`.

**Dependencies:**
*   `internal/rpg/cluster_semantic.go` (EmbeddingClusterer logic).
*   `internal/rpg/enrich.go` (Summarizer interface).
*   `internal/rpg/paths.go` (LCA logic).

## 📝 Implementation Plan

### Prerequisites
*   Ensure `Summarizer` and `SourceLoader` are available in `main.go` (already true for Campaign 3.7).

### Step-by-Step Implementation

#### Phase 1: Core Logic (Global Clusterer)
1.  **Step 1.A (The Harness):** Create `internal/rpg/cluster_global_test.go`.
    *   *Action:* Create a test that verifies `GlobalEmbeddingClusterer` correctly groups nodes and generates names. Mock the `Summarizer` and `Loader`.
    *   *Goal:* Define expected behavior (Map[Name] -> Nodes).
2.  **Step 1.B (The Implementation):** Create `internal/rpg/cluster_global.go`.
    *   *Action:* Implement `GlobalEmbeddingClusterer` struct.
        *   **Fields:** `Inner Clusterer`, `Summarizer Summarizer`, `Loader SourceLoader`.
        *   **Logic:**
            1.  Call `Inner.Cluster(nodes, "root")`.
            2.  For each resulting cluster:
                *   Calculate Centroid.
                *   Select top 5 nodes closest to centroid.
                *   Read snippets using `Loader` (or `atomic_features`).
                *   Call `Summarizer.Summarize(snippets)` to get `Name`.
                *   Store in result map with `Name` as key.
    *   *Detail:* Handle cases where summarization fails (fallback to generic name).
3.  **Step 1.C (The Verification):** Run the test.
    *   *Action:* `go test ./internal/rpg/... -v`
    *   *Success:* Test passes.

#### Phase 2: Builder Integration
1.  **Step 2.A (Refactor Builder):** Modify `internal/rpg/builder.go`.
    *   *Action:* Update `buildGlobal` method.
    *   *Change:*
        *   Current: `domainName := GenerateDomainName(lca, nodes)`
        *   New: Check if `originalKey` (from clusterer) is a "generated ID" (e.g., starts with `root-cluster-`).
            *   If YES: Fallback to `GenerateDomainName(lca, nodes)`.
            *   If NO (it's a semantic name like "Auth"): Use `originalKey` as `domainName`.
    *   *Goal:* Allow the `GlobalClusterer` to dictate the domain name.

#### Phase 3: CLI Wiring & Execution
1.  **Step 3.A (Update Main):** Modify `cmd/graphdb/main.go`.
    *   *Action:* In `handleEnrichFeatures`:
        *   Instantiate `GlobalEmbeddingClusterer`.
        *   Configure it with the existing `VertexSummarizer` and `SourceLoader`.
        *   Assign it to `rpg.Builder{GlobalClusterer: ...}`.
        *   **Crucial:** This effectively enables Global Discovery by default, as `Builder` checks for `GlobalClusterer != nil`.
2.  **Step 3.B (Verification):** Run E2E.
    *   *Action:* Run `enrich-features` on a sample repo (e.g., `test/fixtures`).
    *   *Check:* Inspect `rpg.jsonl`.
        *   Are domain names semantic (e.g., "Math Operations")?
        *   Are functions grouped correctly?
        *   Is `scope_path` set correctly (LCA)?

#### Phase 4: Legacy Cleanup (Removal)
1.  **Step 4.A (Remove Code):** Delete `internal/rpg/discovery.go`.
    *   *Action:* `rm internal/rpg/discovery.go`
    *   *Goal:* Eliminate the `DirectoryDomainDiscoverer` implementation.
2.  **Step 4.B (Remove Tests):** Delete `internal/rpg/discovery_test.go`.
    *   *Action:* `rm internal/rpg/discovery_test.go`
    *   *Goal:* Remove associated tests.
3.  **Step 4.C (Verify Clean Compilation):** Run build.
    *   *Action:* `go build ./...`
    *   *Goal:* Ensure no other parts of the system were implicitly relying on `DirectoryDomainDiscoverer`.
    *   *Note:* `rpg.Builder` should now rely *exclusively* on `GlobalEmbeddingClusterer` for discovery.

### Testing Strategy
*   **Unit Tests:** Focus on `GlobalEmbeddingClusterer` correctly identifying centroids and calling the summarizer.
*   **Integration:** `enrich-features` should produce a valid graph with `Feature` nodes representing Latent Domains.

## 🎯 Success Criteria
*   `rpg.jsonl` contains "Domain" features with semantic names (generated by LLM) instead of directory names.
*   Domains have valid `scope_path` (LCA).
*   Functions are clustered by meaning, not just by folder.
*   `DirectoryDomainDiscoverer` is completely removed from the codebase.
*   `GlobalEmbeddingClusterer` is the sole discovery mechanism.
